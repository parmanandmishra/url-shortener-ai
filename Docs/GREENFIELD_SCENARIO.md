# Greenfield Development Scenario

## Objective

This section demonstrates how Generative AI was used while developing a completely new application from scratch.

The URL Shortener application did not have any existing codebase. Therefore, the implementation followed a Greenfield software development approach where AI assisted throughout the Software Development Lifecycle (SDLC).

---

# Engineering Approach

The development process followed these stages:

Requirement Analysis

↓

Task Decomposition

↓

Architecture Design

↓

AI Assisted Development

↓

Human Review

↓

Testing

↓

Documentation

---

# AI Assisted Workflow

## Step 1 – Requirement Understanding

AI was used to summarize the problem statement and identify the functional and non-functional requirements.

Outcome

- Functional requirements identified
- Non-functional requirements documented
- Risks captured
- Assumptions documented

Human Validation

Reviewed and refined before documentation.

---

## Step 2 – Task Decomposition

Instead of asking AI to build the entire application, the problem was divided into smaller engineering tasks.


Outcome

AI generated implementation backlog including

- Entity
- Repository
- Service
- Controller
- Database
- Tests

Human Validation

Tasks reordered based on implementation priority.

---

## Step 3 – Architecture Design

AI was used for architecture brainstorming.


Generated

- Layered Architecture
- REST APIs
- Database Design

Human Validation

Architecture simplified for assessment while maintaining scalability.

---

## Step 4 – Code Generation

AI accelerated repetitive implementation tasks.

Used for

- DTOs
- Repository
- Controller
- Swagger
- Unit Tests

Business logic was reviewed manually before acceptance.

---

## Step 5 – Testing

AI generated

- Unit Tests
- Mock Objects
- Edge Cases

Tests were reviewed and modified before execution.

---

# Engineering Decision

AI was intentionally used for repetitive engineering work while architecture, business logic and final engineering decisions remained under human control.

This ensured high productivity without compromising software quality.
